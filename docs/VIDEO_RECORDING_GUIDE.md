# 备用演示视频录制指南

目标是录制一版 3-5 分钟视频，面试现场网络不稳定或平台冷启动时可以直接播放。

## 录制设置

- 分辨率：1920x1080 或 1280x720。
- 浏览器缩放：100%。
- 页面语言：中文。
- 录制范围：只录浏览器窗口，避免暴露桌面文件和账号信息。
- 模型模式：优先使用无 Key 离线兜底，保证录制稳定；如要展示真实模型，只展示 LLM 审计结果，不展示密钥。

## 录制前准备

1. 启动本地服务：

```powershell
$env:DATABASE_URL = "sqlite:///:memory:"
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

2. 打开交付演示台：

```text
http://127.0.0.1:8000/demo
```

3. 跑一次验收脚本：

```powershell
.\scripts\final_acceptance.ps1 -BaseUrl http://127.0.0.1:8000
```

4. 清理浏览器多余标签页，只保留首页、演示台和结果页。

## 推荐镜头脚本

### 0:00-0:30 产品定位

打开首页，说明它是 DBA 变更交付系统，不是聊天机器人。指出首页的交付就绪度、案例数、确认进度、签收进度和离线兜底记录。

### 0:30-1:10 演示案例

进入 `/demo`，展示推荐 DB2 索引变更案例。说明输入包含数据库类型、目标系统、变更目标、SQL、表结构和约束条件。

### 1:10-2:10 生成交付包

点击“一键生成交付包”或“一键完整闭环”。进入结果页后依次展示 6 类交付物：风险评估、执行 Runbook、回滚方案、前置检查 SQL、验收清单和沟通摘要。

### 2:10-3:00 审计与人工复核

展示右侧 LLM 调用审计，说明真实模型失败或无 Key 时会记录原因并进入离线兜底。编辑一份交付物，保存并确认，展示版本记录和最近内容变化。

### 3:00-4:00 签收与导出

点击“确认全部交付物”，再点击“签收交付包”。导出 Markdown 或 PDF，说明导出文档包含封面、目录、完成度、签收、版本和审计记录。

### 4:00-4:40 运行状态

打开 `/ops`，说明部署后可以用这个页面和 `/healthz` 检查数据库、模型模式、案例统计和交付统计。

### 4:40-5:00 技术收尾

用一句话收束：

> 这个项目用 FastAPI、Jinja2、SQLAlchemy 和 OpenAI-compatible 适配层做出一个可试运行、可审计、可导出的 DBA AI 变更交付闭环。

## 交付文件命名

推荐命名：

```text
dba-changeops-ai-workbench-demo-2026-06.mp4
```

录制完成后，把视频上传到可访问的位置，并在 README 顶部补充链接。公开视频不要包含真实数据库地址、真实业务数据、真实密钥或公司内部系统名称。
